# 二手闲置社交交易平台
# 第一版前端信息架构与页面清单草案（uni-app / Vue3）

> 适用阶段：前端骨架期
>
> 目标：先定页面骨架、路由分组和目录边界，方便后续子 agent 按模块并行实现。
>
> 原则：
> - 先覆盖主流程，不展开复杂视觉设计
> - 移动端优先，兼容 uni-app / Vue3
> - 底部导航清晰，业务页面围绕交易闭环组织
> - 登录、订单、钱包、认证、后台入口单独分组

---

## 1. 信息架构总览

### 1.1 首期核心用户路径

1. 浏览首页/分类
2. 进入商品详情
3. 发起咨询或下单
4. 进入 IM 聊天沟通
5. 查看订单状态
6. 查看钱包余额与流水
7. 进行实名认证
8. 进入个人中心处理我的发布、订单、消息、设置
9. 后台入口独立进入管理页

### 1.2 首期必须有的页面域

- 底部导航页
  - 首页
  - 分类
  - 发布
  - 消息
  - 我的
- 商品域
  - 商品详情

  - 搜索结果 / 筛选结果
- IM 域
  - 会话列表
  - 单聊窗口
  - 系统通知
- 订单域
  - 订单列表
  - 订单详情
  - 订单确认 / 取消 / 完成
- 钱包域
  - 钱包首页
  - 收支流水
  - 提现申请

## 2. 底部导航建议

### 2.1 TabBar 结构

建议首期采用 5 个 Tab：

1. `首页`：内容推荐、搜索入口、快捷分类、热门商品
2. `分类`：按类目浏览、筛选、专题入口
3. `发布`：发闲置、上传图片、填写商品信息
4. `消息`：会话、系统消息、交易通知
5. `我的`：个人中心、订单、钱包、认证、设置

### 2.2 Tab 页面职责

- `首页`：承接发现和转化
- `分类`：承接明确需求查找
- `发布`：承接卖家发品主流程
- `消息`：承接沟通与通知
- `我的`：承接账户与资产管理

### 2.3 Tab 入口约束

- `商品详情`、`聊天详情`、`订单详情`、`钱包`、`认证` 不放入 Tab，作为二级页面进入
- `登录注册`、`后台入口` 不放入 Tab，独立分组
- 发布页可作为 Tab，但内部是完整发布流程页，不建议再嵌套太多 Tab

## 3. 初始路由分组建议

### 3.1 路由分组原则

按“用户主流程 + 独立功能域”分组：

- `tabbar`：底部导航页
- `product`：商品详情、图片预览、搜索筛选
- `chat`：IM 会话和聊天
- `order`：订单列表和详情
- `wallet`：钱包与提现
- `auth`：登录、注册、验证码、认证引导
- `user`：个人中心相关子页

## 4. 页面清单草案

### 4.1 首页 `tabbar/home/index`

**页面目标**：发现商品、快速搜索、进入详情。

**首期模块**
- 顶部搜索栏
- 城市 / 校区切换入口（可先做静态）
- Banner / 活动位
- 快捷分类入口
- 推荐商品流
- 热门卖家 / 附近好物（可选）
- 空态 / 加载态

**跳转**
- 商品详情
- 搜索结果页
- 分类页
- 发布页

---

### 4.2 分类 `tabbar/category/index`

**页面目标**：按类目查找商品。

**首期模块**
- 左侧一级分类
- 右侧二级分类 / 商品卡片
- 筛选入口：价格、成色、距离、发布时间
- 最近浏览入口（可选）

**跳转**
- 分类商品列表
- 商品详情

---

### 4.3 发布 `tabbar/publish/index`

**页面目标**：快速发布闲置商品。

**首期模块**
- 商品标题
- 商品类目选择
- 图片上传
- 商品描述
- 价格
- 成色
- 交易方式
- 联系方式或默认 IM 联系
- 发布确认

**流程建议**
- 先做单页表单版
- 如后续复杂，再拆成分步发布

**跳转**
- 发布成功结果页
- 我的发布

---

### 4.4 消息 `tabbar/message/index`

**页面目标**：承接 IM 会话和交易通知。

**首期模块**
- 会话列表
- 系统通知入口
- 订单通知入口
- 未读角标

**跳转**
- 聊天窗口
- 通知详情（可先复用列表页）

---

### 4.5 我的 `tabbar/me/index`

**页面目标**：个人中心总入口。

**首期模块**
- 头像、昵称、认证状态
- 我的订单入口
- 我的发布入口
- 我的钱包入口
- 我的收藏入口
- 我的认证入口
- 设置入口

**跳转**
- 个人资料
- 订单列表
- 钱包首页
- 认证页
- 我的发布
- 设置

## 5. 重点业务页面

### 5.1 商品详情 `product/detail/index`

**页面目标**：详情展示、促成咨询或下单。

**首期模块**
- 商品图集
- 标题、价格、成色、发布信息
- 卖家信息卡
- 商品描述
- 交易说明
- 收藏 / 分享 / 私聊 / 立即购买按钮
- 推荐商品

**关键能力**
- 跳转聊天窗口
- 跳转订单确认
- 跳转图片预览

---

### 5.2 IM 聊天 `chat/conversation/index`

**页面目标**：买卖双方沟通。

**首期模块**
- 顶部对话对象信息
- 消息列表
- 文本输入框
- 图片发送入口（首期可选）
- 商品卡片引用
- 系统提示消息
- 已读状态 / 时间分割

**配套页面**
- 会话列表 `chat/session-list/index`

**注意**
- 首期只做单聊
- 群聊、语音、红包等不进入首版

---

### 5.3 订单 `order/*`

**页面目标**：完成下单到履约状态查看。

**页面清单**
- `order/list/index`
  - 全部 / 待支付 / 待发货或待交付 / 待完成 / 已完成 / 已取消
- `order/detail/index`
  - 商品快照
  - 买卖双方信息
  - 订单状态流转
  - 支付信息
  - 操作按钮
- `order/confirm/index`
  - 确认下单
  - 确认金额与交易方式
- `order/result/index`
  - 成功 / 失败 / 取消结果页

**首期必须状态**
- 待支付
- 待确认
- 已完成
- 已取消

---

### 5.4 钱包 `wallet/*`

**页面目标**：查看余额与资金记录。

**页面清单**
- `wallet/index`
  - 可用余额
  - 冻结金额
  - 提现入口
  - 收支摘要
- `wallet/ledger/index`
  - 流水列表
  - 入账 / 出账 / 冻结 / 解冻状态
- `wallet/withdraw/index`
  - 提现账户信息
  - 提现金额
  - 提现申请提交

**首期注意**
- 钱包页面只做展示与申请，不在前端假设资金规则
- 所有金额单位统一为元，展示层避免多精度误差

---

### 5.5 认证 `auth/realname-guide/index`

**页面目标**：引导用户完成实名认证。

**页面清单**
- 认证引导页
- 认证信息填写页
- 认证结果页

**首期模块**
- 认证必要性说明
- 姓名、证件号、手机号
- 提交状态
- 审核中 / 通过 / 驳回原因

---

### 5.6 后台入口 `admin/entry/index`

**页面目标**：给运营/管理员保留独立入口。

**首期形态**
- 作为独立入口页，不放 Tab
- 可从 `我的` 页按角色显隐跳转
- 首版只保留登录入口和后台壳页

**页面清单**
- `admin/entry/index`
  - 后台登录 / 口令入口
- `admin/home/index`
  - 后台首页壳
  - 预留审核、商品、订单、用户入口

## 8. 第一版必须控制的范围

首期只做这些，不扩展：

- 不做复杂社交动态流
- 不做群聊、语音、视频通话
- 不做优惠券、积分、复杂营销中心
- 不做复杂搜索推荐算法
- 不做多角色后台完整运营面板
- 不做复杂装修页和强视觉首页

## 9. 可直接交付给子 agent 的结论

### 9.1 页面骨架优先级

- P0：TabBar 5 页 + 商品详情 + 会话列表 + 聊天窗口 + 订单列表/详情 + 钱包首页 + 认证页 + 后台入口
- P1：搜索结果、分类列表、发布结果、个人资料、设置、收藏、提现
- P2：系统通知详情、结果页、协议页、空态页

### 9.2 路由分组结论

- `tabbar`：承载主导航
- `product`：商品详情与搜索浏览
- `chat`：IM 独立域
- `order`：交易状态域
- `wallet`：资金域
- `auth`：登录与认证域
- `user`：个人中心域
- `admin`：后台入口域
- `system`：兜底页

### 9.3 落地原则

- 先壳后细
- 先路由后交互
- 先主流程后边角功能
- 先单页可跑通，再拆子组件
