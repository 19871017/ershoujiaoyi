# 小原圈上线测试搭建说明

> 项目目录：`/mnt/c/Users/WangYu/Desktop/esxz-sj-jypt`  
> 适用目标：本地/内网/测试服务器上线测试，覆盖后端 API、用户端 H5、管理后台。  
> 重要原则：上线测试环境必须使用真实后端、真实数据库、真实媒体上传票据、真实后台登录态；不要把 mock/dev headers 当成正式测试结果。

## 1. 项目结构

```text
esxz-sj-jypt/
├── backend/   # Spring Boot 后端 API，Maven 项目
├── frontend/  # uni-app 用户端 H5
└── admin/     # Vue3 + Vite 独立管理后台
```

## 2. 基础环境要求

### 2.1 必需软件

- JDK：17 或以上
- Maven：3.8 或以上
- Node.js：18 或以上，建议 20+
- npm：随 Node 安装
- MySQL：8.0 或兼容版本
- Nginx：正式测试服务器建议使用，用于反向代理和静态资源托管

### 2.2 推荐端口

- 后端 API：`8080` 或 `18080`
- 用户端 H5：Nginx 静态目录或临时预览端口 `4174`
- 管理后台：Nginx 静态目录或临时预览端口 `5173`

> 注意：当前源码默认后端端口为 `SERVER_PORT`，未设置时使用 `8080`。管理后台源码默认 API 地址为 `VITE_API_BASE`，未设置时指向 `http://localhost:18080`。上线测试时必须显式配置 API 地址，避免前端连错端口。

### 3.2 后端环境变量

进入后端目录：

```bash
cd /mnt/c/Users/WangYu/Desktop/esxz-sj-jypt/backend
```

设置环境变量示例：

```bash
export SERVER_PORT=18080
export DB_URL='jdbc:mysql://127.0.0.1:3306/esxz_sj_jypt?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true'
export DB_USERNAME='你的数据库用户名'
export DB_PASSWORD='你的数据库密码'
export DB_DRIVER='com.mysql.cj.jdbc.Driver'
export DB_INIT_MODE=always
export DB_SCHEMA_LOCATIONS='classpath:db/schema.sql'
export DB_DATA_LOCATIONS='classpath:db/data.sql'
```

说明：

- 首次搭建测试环境可使用 `DB_INIT_MODE=always` 初始化表结构与基础数据。
- 已有真实数据后，不建议长期使用 `always`，避免误判初始化行为；可改为 `never`，后续由迁移脚本或人工 SQL 管理变更。
- 不要把数据库密码、支付密钥、对象存储密钥写入 Git；通过服务器环境变量或部署平台 Secret 管理。

### 3.4 后端关键接口

用户端：

- `POST /api/auth/login`：客户注册/登录；新手机号首次登录会创建用户账号和资料。

- `POST /api/orders`：创建订单。
- `POST /api/orders/{orderNo}/pay`：支付订单。
- `POST /api/after-sales`：发起售后。
- `GET/POST /api/wallet/**`：钱包、提现账号、提现申请。
- `POST /api/chat/messages`：聊天消息。
- `POST /api/audit/reports`：举报投诉。

管理后台：

- `POST /api/admin/session/login`：后台登录。
- `GET /api/admin/session/me`：刷新后台登录态。
- `POST /api/admin/session/logout`：退出后台。
- `GET /api/admin/dashboard`：运营首页汇总。
- `GET /api/admin/users`、`GET /api/admin/users/{userId}`：用户检索与详情。
- `GET /api/admin/audit`、`GET /api/admin/audit/{auditNo}`：认证/举报/提现等审核列表与详情。
- `POST /api/admin/audit/{auditNo}/approve|reject`：审核通过/驳回。
- `GET /api/admin/orders`、`GET /api/admin/orders/{orderNo}`：订单管理。
- `GET /api/admin/after-sales`、`POST /api/admin/after-sales/{afterSalesNo}/approve|reject`：售后管理。
- `GET /api/admin/withdrawals`：提现复核。
- `GET/POST /api/admin/location/config`：定位配置。
- `GET /api/admin/audit-logs`：后台操作日志。

Nginx 可将：

- `/h5/` 指向 `frontend/dist/build/h5/`
- `/assets/` 指向 `frontend/dist/build/assets/`

## 6. Nginx 部署示例

以下为示意，请按实际域名、证书路径和服务器目录调整：

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:18080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name h5.example.com;
    root /var/www/xiaoyuan/frontend/dist/build;

    location /h5/ {
        try_files $uri $uri/ /h5/index.html;
    }

    location /assets/ {
        try_files $uri =404;
    }
}

server {
    listen 80;
    server_name admin.example.com;
    root /var/www/xiaoyuan/admin/dist;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

正式测试建议启用 HTTPS，并将 HTTP 301 跳转到 HTTPS。

## 8. 上线测试安全要求

- 用户端生产构建必须关闭：`VITE_ENABLE_MOCK_DATA=false`、`VITE_ENABLE_DEV_HEADERS=false`、`VITE_ENABLE_LAN_API_FALLBACK=false`。
- 管理后台所有受保护接口必须携带：`X-User-Id` 与 `X-Admin-Session`。
- 不允许在前端保存后台密码、数据库密码、支付密钥。
- 敏感媒体必须通过后端上传票据签发后再保存，场景包括：`VIDEO_IDENTITY`、`PRODUCT_IMAGE`、`COMMUNITY_IMAGE`、`AFTER_SALES_EVIDENCE`、`REPORT_EVIDENCE`、`CHAT_IMAGE`。
- 生产/上线测试环境不要使用未授权的外链图片、占位图、伪造 URL 持久化到数据库。
- 支付回调、提现、售后、认证审核必须以服务端状态为准，不能以前端按钮点击结果为准。
- 日志与后台审计记录必须脱敏手机号、银行卡号、密钥、token。

## 9. 常见问题

### 9.1 管理后台登录后接口 403

检查：

- 后端是否启动在 `VITE_API_BASE` 指向的地址。
- 登录接口是否返回 `sessionId`。
- 后台请求是否携带 `X-User-Id` 和 `X-Admin-Session`。
- 当前用户是否在 `admin_user_permission` 中有对应权限。
- `admin_session` 是否过期或 revoked。

### 9.2 H5 手机访问空白

检查：

- `frontend/dist/build/h5/index.html` 是否存在。
- `/assets/` 是否能访问到真实 JS/CSS。
- 若部署到 `/h5/` 子路径，是否已复制 `dist/build/h5/assets` 到 `dist/build/assets` 或配置 Nginx alias。
- 浏览器控制台是否有 404 或 API 跨域错误。

### 9.3 客户登录成功但接口仍提示未登录

检查：

- 用户端是否保存 `accessToken`。
- 请求头是否携带 `Authorization: Bearer <token>`。
- 后端用户鉴权解析是否与当前 token 格式匹配。
- 上线测试构建是否误关了真实 API 地址。

### 9.4 首次数据库没有表

检查：

- `DB_INIT_MODE` 首次是否设置为 `always`。
- `DB_SCHEMA_LOCATIONS` 是否为 `classpath:db/schema.sql`。
- 数据库账号是否有建表权限。
- MySQL 连接串时区、编码是否正确。
